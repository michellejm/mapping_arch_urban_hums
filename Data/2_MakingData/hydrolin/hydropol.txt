    <descript>
      <abstract>The hydro polygon/arc coverages were created using TIGER/LINE 2000 shapefile data gathered from ESRI's Geography Network. The individual county hydrography line shapefiles were processed into Arc/Info coverages and then appended together to create complete state coverages. They were then edited to remove unwanted features, leaving a state-by-state database of both important and navigable water features. Attributes were added to denote navigable features and names. Also, an attribute was added to the polygons to denote which were water and which were land features. The state databases were then appended together to create a single, nationwide hydrography network containing named arcs and polygons. These features also contain a state FIPS. Because some of the hydro features are represented by lines instead of polygons, the complete hydro dataset consists of 2 shapefiles, one for lines and one for polygons. They must be used together to paint a complete picture.</abstract>
      <purpose>The BTS desires a single nationwide database that has polygons/attributes for major rivers and lakes, is consistent across state lines, and has the most accurate data available. This database will be used primarily for map production, basic queries, and will be distributed as part of the National Transportation Atlas Database (NTAD) 2014.</purpose>
      <supplinf>The BTS HYDRO Database contains TIGER 2000 geometry and attributes. UTCTR flagged this database to represent major water features and edited the geometry to insure polygons would fill for cartographic representation. To find out more about TIGER/Line files and other Census TIGER data base derived data sets visit http://www.census.gov/geo/www/tiger.</supplinf>

      <procstep>
        <procdesc>When UTN processed the TIGER dataset for State, County and Hydrographic (lines and polygons), a 3-mile buffer was created around the Texas shorelines. This buffer removed the island off the coast of Texas. To correct this error, BTS used ARC/INFO to remove the 3-mile buffer to capture the islands along Texas. To capture the islands, the state dataset needed to be reselected by selecting: code = 1. This process created a few inaccurate polygons and they were removed. To create the final datasets, the datasets were dissolved and regions created on STFIPS and County FIPS.</procdesc>
        <procdate>20030301</procdate>
      </procstep>
      <procstep>
        <procdesc>1) Downloaded zipped shapefiles from www.geographynetwork.com; 2) Converted the Shapefiles to Coverages; 3) Removed all intermittent water features H02, H12, H22, H32, H42, H60, H71, H72, H80, and H81 from the database coverages; 4) Flagged arcs to be retained in the output coverage based on relative importance and research using the Internet and state maps; 5) Added attributes to polygons to denote name description and land/water. These attributes were then populated by user input using mapping resources such as state and county maps; 6) Added attribute to both arcs and polygons to denote navigable waterways. For single line features, the arcs were flagged. If a feature contained polygons, the polygons only were flagged; 7) States were then appended, extra items dropped, data released.</procdesc>
